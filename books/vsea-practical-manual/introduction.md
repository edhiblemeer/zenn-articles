# VSEA実践マニュアルへようこそ

このマニュアルは、**VSEA (Vertical Slice Evolutionary Architecture)** という新しい開発手法の実践ガイドです。

## このマニュアルで学べること

- VSEAの基本原則と核心概念
- フェーズ別の詳細な実施手順
- 縦スライスの設計方法
- 実践で遭遇する課題と解決策
- すぐに使えるテンプレートとチェックリスト

## 対象読者

- 業務アプリケーション開発者
- アジャイル開発に興味がある方
- 技術的負債を減らしたい方
- 新しい開発手法を学びたい方

## 実践プロジェクト

このマニュアルは、実際のプロジェクト(スケジュール自動登録システム)で**1日(実働8時間)**で開発した経験を基に作成されています。

開発内容:
- CSVファイルからスケジュールデータを読込み、CMSに自動登録
- スタッフマスタ照合、週単位登録、エラーハンドリング
- Python + tkinter + Playwright

## マニュアルの使い方

### 初めての方
1. 第1章「基本原則」を読む
2. 第2-5章「フェーズ別ガイド」で実施手順を理解
3. 小規模プロジェクトで実践

### 実践中の方
- チェックリスト(第8章)で進捗確認
- テンプレート(第9章)をコピーして使用
- 課題に遭遇したら第7章を参照

### 振り返り時
- FAQ(第10章)で疑問を解消
- 他のプロジェクトへの適用を検討

## VSEAの特徴

従来手法との比較:

| 項目 | 従来手法 | VSEA |
|-----|---------|------|
| リスク管理 | 最後まで動かない | 常に動くものがある |
| 変更対応 | 大規模リファクタリング | 追加改善で対応 |
| フィードバック | 後工程で発覚 | 各スライスで即座に |
| 開発速度 | 遅い | 爆速 |

## 開発速度の実績

実際のプロジェクトでの開発時間:

| フェーズ | 時間 |
|---------|------|
| UX整合設計 | 30分 |
| スケルトン作成 | 30分 |
| 縦スライス実装 | 5.5時間 |
| 肉付け | 1.5時間 |
| パッケージング | 30分 |
| **合計** | **8時間** |

たった1日で実稼働可能なアプリが完成しました。
