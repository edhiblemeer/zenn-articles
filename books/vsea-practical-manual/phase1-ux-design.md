# フェーズ①: UX整合設計

## 目的

ユーザー体験の設計とプロジェクト全体像の把握

## 成果物

1. ユーザーストーリー(5-10件)
2. 画面遷移図
3. エラーケース一覧(5-10件)
4. GUIモックアップ

## 実施手順

### Step 1: ユーザーストーリー定義

**テンプレート**:
```
タイトル: [誰が][何を][なぜ]
説明: [背景][期待される動作][受け入れ基準]
優先度: 高/中/低
```

**実例**:
```markdown
## ストーリー1: CSV読込
- 誰が: ユーザー
- 何を: CSVファイルを選択して読込む
- なぜ: 手入力の手間を削減するため

背景:
- 現在は手動でデータ入力している
- 月に1回、約500件のデータ入力作業が発生

期待される動作:
1. ファイル選択ダイアログが表示される
2. CSVファイルを選択できる
3. 読込結果が表示される

受け入れ基準:
- UTF-8とShift-JISに対応
- 必須カラムの検証
- エラー時は分かりやすいメッセージ表示

優先度: 高
```

### Step 2: 画面遷移設計

**実例**:
```
[起動画面] → (ファイル選択) → [検証結果画面]
                             ↓ (検証OK)
                           [登録確認画面]
                             ↓ (登録開始)
                           [進捗表示画面]
                             ↓ (完了)
                           [結果サマリー画面]
```

### Step 3: エラーケース特定

| エラーケース | 原因 | 対応 | 優先度 |
|------------|------|------|--------|
| ファイル読込失敗 | 不正なフォーマット | エラーメッセージ表示 | 高 |
| 必須カラム欠損 | CSVカラム不足 | 不足カラム名を表示 | 高 |
| スタッフ名不一致 | マスタ未登録 | 3択ダイアログ表示 | 中 |
| ネットワークエラー | CMS接続失敗 | リトライ提案 | 中 |

### Step 4: GUIモックアップ

簡易テキストモックアップでOK:

```
┌─────────────────────────────────────┐
│ アプリ名 v1.0                       │
├─────────────────────────────────────┤
│ CSVファイル: [選択...] [スタッフ更新]│
│                                     │
│ 検証結果:                           │
│ ✓ フォーマット: OK                  │
│ ✓ 総件数: 540件                     │
│                                     │
│ [登録開始]                          │
└─────────────────────────────────────┘
```

## チェックリスト

- [ ] 各ストーリーに受け入れ基準がある
