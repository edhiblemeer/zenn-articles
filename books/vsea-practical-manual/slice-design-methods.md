# 縦スライスの設計方法

## INVEST原則

- **I**ndependent: 独立している
- **N**egotiable: 交渉可能(柔軟)
- **V**aluable: 価値がある
- **E**stimable: 見積もり可能
- **S**mall: 小さい(1-2時間)
- **T**estable: テスト可能

## 切り方パターン

### パターン1: 機能単位

```
Slice 1: ユーザー登録機能
Slice 2: ログイン機能
```

### パターン2: ユーザーストーリー単位

```
Slice 1: "ユーザーとして、CSVファイルを読み込んで検証したい"
Slice 2: "ユーザーとして、検証結果をCMSに登録したい"
```

### パターン3: MVPベース

```
Slice 1: MVP(最小限の価値提供)
Slice 2: エラーハンドリング追加
Slice 3: UX改善
```
